# Generate Change Log Title and Description

## Rule

1. Read the text after the word "Content".
2. Clean it up and make it enjoyable for a 10 year old read.
3. Using markdown formatting as shown in the public/data/changelog.json
4. Make sure to include line breaks correctly.
5. Create code blocks for the title and description.
6. Generate a DateTime stamp
7. Make sure the description codeblock uses "\n" for line breaks and "\n\n".
8. description should all be ineline with formatting.
